SAÉ 1.05 Compétence RT3 · Coeff. 35

Traiter des données

Développement en binôme d'un script Python pour collecter automatiquement des données via une API REST OpenStreetMap, les traiter statistiquement et produire un rapport exporté au format Markdown/HTML.

Python API REST JSON / XML OpenStreetMap Markdown requests Statistiques
01

Contexte & Objectifs

Problématique professionnelle : Le professionnel R&T est régulièrement amené à traiter des données issues du système d'information de l'entreprise (état des équipements, utilisateurs, logs réseau). Ces données brutes doivent être traitées, nettoyées et présentées de façon synthétique, idéalement de manière automatisée via des scripts.

Mission confiée : En binôme, créer un programme Python capable de : (1) récolter des données via l'API REST OpenStreetMap/Overpass, (2) mettre en forme les données selon les informations souhaitées, (3) effectuer des traitements statistiques, (4) exporter les résultats au format Markdown, puis en HTML via la bibliothèque Python markdown.

Organisation : Projet en binôme sur 20h en autonomie, avec deux points d'étape évalués (mi-projet et fin de projet). Note finale = moyenne entre note de projet et note de suivi.

02

Compétences techniques mobilisées

  • API REST et protocole HTTP : Compréhension du modèle requête/réponse REST, codes de statut HTTP (200, 404, 500…), paramètres d'URL, utilisation de la bibliothèque requests pour les appels HTTP en Python.
  • Overpass API (OSM) : Langage de requête Overpass QL pour interroger la base de données OpenStreetMap, extraction de nœuds et de relations géographiques avec filtres par clé/valeur (amenity, shop, etc.).
  • Formats JSON et XML : Parsing et manipulation de structures imbriquées avec json.loads() et xml.etree.ElementTree en Python.
  • Programmation Python : Listes, dictionnaires, fonctions, gestion de fichiers, calculs statistiques (moyenne, médiane, fréquences), organisation du code en modules.
  • Format Markdown : Syntaxe de balisage légère (titres, gras, listes, tableaux), compilation en HTML via la bibliothèque Python markdown.
03

Ma contribution & tâches réalisées

Choix du jeu de données : Mon binôme et moi avons choisi d'analyser la répartition des équipements de restauration (restaurants, cafés, fast-food) dans la ville de Caen. Ce sujet nous permettait d'exploiter des données riches et structurées sur OSM, avec des attributs variés (nom, type de cuisine, horaires).

Module de collecte : J'ai développé le module Python de collecte en écrivant une requête Overpass QL pour extraire tous les nœuds avec le tag amenity=restaurant ou amenity=cafe dans un rayon de 5 km autour du centre de Caen. La bibliothèque requests effectue l'appel HTTP, et la réponse JSON est parsée et filtrée.

Module de traitement : Mon binôme a développé les fonctions de statistiques descriptives (répartition par type de cuisine, nombre d'établissements par quartier) et j'ai créé le module de génération du rapport Markdown avec tableaux et listes formatés.

Export HTML : Le script final lit le fichier .md généré et utilise la bibliothèque Python markdown pour produire un fichier HTML autonome avec un CSS inline minimal pour la lisibilité.

Points d'étape : Au premier point d'étape, nous avions le module de collecte fonctionnel et les premières statistiques. Au second, le rapport complet Markdown et HTML était généré automatiquement.

04

Traces & livrables

  • Script Python commenté — module de collecte (collect.py), traitement (stats.py), export (report.py)
  • Fichier rapport généré au format Markdown (rapport.md)
  • Fichier HTML produit par compilation (rapport.html)
  • Évaluations des deux points d'étape intermédiaires
05

Analyse réflexive

Difficultés rencontrées : La structure JSON renvoyée par l'API Overpass était plus complexe que prévu — les données sont encapsulées dans un tableau elements avec des nœuds, des chemins et des relations. Il fallait filtrer uniquement les nœuds avec des coordonnées valides. Un autre problème était la gestion des données manquantes : beaucoup d'établissements n'avaient pas de tag cuisine, ce qui nécessitait de traiter les cas None sans planter le script.

Solutions apportées : J'ai d'abord testé mes requêtes Overpass directement sur overpass-turbo.eu avant de les intégrer en Python, ce qui m'a permis de visualiser la structure des données retournées. Pour les valeurs manquantes, j'ai ajouté une valeur par défaut "Non renseigné" qui permettait quand même d'inclure ces établissements dans les statistiques.

Ce que j'ai appris : Cette SAÉ m'a permis de comprendre concrètement le modèle REST et son usage dans les systèmes d'information, d'automatiser le traitement de données en Python, et de produire des rapports lisibles à la fois par des humains et des machines (Markdown/HTML). La coordination en binôme sur un projet de code m'a appris à découper le travail en modules indépendants et à définir des interfaces claires entre eux.

Positionnement personnel : Cette SAÉ m'a donné confiance en ma capacité à développer des scripts utiles. Je maîtrise les API REST et le traitement JSON, mais je dois encore progresser sur la structuration du code Python en modules réutilisables et sur les tests unitaires pour fiabiliser mes scripts.

SAÉ 1.04 Compétence RT3 · Coeff. 8

Se présenter sur Internet

Concevoir et réaliser un site web multipage en HTML/CSS dans le cadre de la ressource R1.15 — Gestion de projet. Site responsive, conforme aux standards W3C et WCAG 2.0 AA, avec animations CSS et hébergement GitHub Pages. Ce portfolio est le livrable de cette SAÉ.

HTML5 / CSS3 Flexbox / CSS Grid Responsive Design WCAG 2.0 AA Animations CSS GitHub Pages Gestion de projet
01

Contexte & Objectifs

Projet : Dans le cadre de la SAÉ 1.04 et de la ressource R1.15 «Gestion de projet : Maîtriser les bases», réaliser un site web multipage au thème libre. J'ai choisi de créer un portfolio académique présentant l'ensemble de mes SAÉ du semestre 1, ce qui constituait à la fois un livrable utile et un exercice complet de développement web.

Cahier des charges : Minimum 3 pages, charte graphique cohérente, validation W3C, responsive (mobile/tablette/ordinateur), animations CSS, hébergement GitHub Pages, commits porteurs de sens, README.md détaillé.

Notation : Design et UX/UI (4 pts), contenus et rédactionnel (4 pts), respect du cahier des charges (6 pts), livrables R1.15 — WBS, rétroplanning, tableau de suivi (6 pts).

02

Exigences techniques ciblées

  • Structure HTML sémantique : Utilisation des balises HTML5 (<nav>, <main>, <section>, <article>, <footer>), arborescence cohérente avec 8 pages interconnectées.
  • CSS avancé : CSS Grid et Flexbox pour la mise en page, variables CSS (:root) pour la charte graphique, media queries pour le responsive design.
  • Animations CSS : @keyframes pour les animations d'entrée (fadeUp), transitions sur les cartes et la navigation, effet de défilement sur la barre de navigation.
  • Accessibilité WCAG 2.0 AA : Ratio de contraste texte/fond vérifié (minimum 4.5:1), attributs aria-label sur les boutons sans texte, navigation au clavier fonctionnelle.
  • Versioning et déploiement : Repository GitHub avec commits atomiques et messages descriptifs, hébergement automatique via GitHub Pages.
03

Ma contribution & tâches réalisées

Conception du design : J'ai opté pour une esthétique sombre et minimaliste inspirée des portfolios de développeurs, avec une palette basée sur des bleus désaturés (#7eb8d4) et des tons chaleureux (#c4a882). La typographie associe Cormorant Garamond (empattements, titres) et DM Mono (monospace, étiquettes), donnant un caractère à la fois technique et élégant.

Architecture des pages : J'ai structuré le site avec une page d'accueil (index.html), une page dédiée aux SAÉ (saes.html) et cinq pages détail (sae-01.html à sae-05.html). La navigation est universelle et accessible depuis toutes les pages via un menu déroulant.

Gestion de projet R1.15 : J'ai créé un WBS décomposant le projet en phases (conception, développement, tests, déploiement), un rétroplanning sur 4 semaines et un tableau Kanban sur Trello pour le suivi des tâches.

Déploiement : Le site est hébergé sur GitHub Pages. J'ai effectué plus de 20 commits avec des messages descriptifs (ex. : « feat: add dropdown nav with SAE links », « fix: improve mobile menu accessibility »).

04

Traces & livrables

  • Site web multipage (8 pages) en HTML/CSS — ce portfolio hébergé sur GitHub Pages
  • Repository GitHub avec historique de commits significatifs (>20 commits)
  • README.md détaillé (présentation du projet, technologies, lien hébergement)
  • WBS du projet (décomposition des tâches)
  • Rétroplanning sur 4 semaines
  • Tableau Kanban Trello — suivi d'avancement
05

Analyse réflexive

Difficultés rencontrées : La mise en place du menu déroulant accessible depuis toutes les pages a été plus complexe que prévu — les interactions CSS pur (:hover) ne fonctionnant pas sur mobile, il a fallu basculer vers une solution JavaScript. La validation W3C a également révélé plusieurs erreurs d'imbrication de balises, notamment autour des éléments <a> contenant des <article>, ce qui n'est pas valide en HTML5.

Solutions apportées : J'ai restructuré le HTML pour envelopper les articles dans des liens plutôt que l'inverse, ce qui a résolu les erreurs W3C. Pour le menu mobile, j'ai implémenté un toggle JavaScript qui gère l'ouverture/fermeture du menu burger avec des animations CSS sur les spans.

Ce que j'ai appris : Cette SAÉ m'a appris le cycle complet de création d'un site web professionnel, de la conception à la mise en ligne. J'ai découvert l'importance des variables CSS pour maintenir une charte cohérente, et la puissance des animations CSS @keyframes pour donner vie aux interfaces. La gestion de projet avec un WBS et un rétroplanning m'a aidé à structurer mon temps et à respecter les délais.

Positionnement personnel : Je me situe à un bon niveau sur RT3 pour le développement web front-end. L'étape suivante est l'apprentissage de JavaScript plus avancé et éventuellement d'un framework CSS ou JS pour des projets plus ambitieux.
